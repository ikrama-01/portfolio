Project Overview

This academic research project analyzes the network performance of RESTful and gRPC API methods using the IGDB API (by Twitch) to fetch real-time video game data including names, screenshots, covers, and more.

It demonstrates how gRPC (with HTTP/2) outperforms REST in terms of speed, payload size, and efficiency, especially for large datasets.

üí° Project Details

Research Methodology

The project followed a structured approach to compare the two protocols:

  • Set up both RESTful and gRPC endpoints to the IGDB API
  • Created identical data requests with varying payload sizes
  • Measured response times and payload sizes across multiple test runs
  • Visualized the results through interactive comparisons

Takeaway & Learning

This project provided valuable insights beyond just frontend development:

  • Understanding network protocols and their performance characteristics
  • Learning about data serialization methods (JSON vs Protocol Buffers)
  • Appreciating the importance of compression and efficient data transmission
  • Gaining experience with OAuth authentication and API management
  • Developing skills in performance testing and benchmarking

Future Scope

The insights from this project have significant implications for future work:

  • gRPC with HTTP/2 is increasingly adopted by major companies like Netflix and Cisco
  • The performance benefits become more pronounced with microservice architectures
  • Understanding these protocols positions developers to create more scalable systems
  • The techniques learned can be applied to real-time applications where performance is critical
