Project Overview

The Metro Management System is a robust backend-driven application that powers metro ticket booking, wallet-based fare payments, and role-based user operations. Built entirely using Java (Spring Boot) and MySQL, it supports secure JWT authentication, admin controls, and smooth user interactions for a modern metro service.

This project lays the groundwork for a scalable metro platform where users can register, log in, recharge wallets, and book tickets ‚Äî while admins manage routes, fares, and monitor usage.

Database Design

The project utilizes a carefully designed relational database schema:

  • users: id, username, email, password, role (admin/user), wallet_balance
  • stations: id, name, code
  • routes: id, source_id, destination_id, fare, distance
  • tickets: id, user_id, source_id, destination_id, fare, created_at

Foreign keys are used for route and ticket associations, wallet_balance is deducted during ticket booking and updated after recharge, and created_at in tickets stores time of booking for future tracking.

Security Implementation

The project features robust security measures:

  • Full JWT-based secure backend with no external auth dependencies
  • Password encryption using industry-standard hashing algorithms
  • Role-based access control for different user types
  • Secure endpoints for sensitive operations like wallet recharges

System Architecture

Built with industry-standard practices:

  • Clean separation of concerns using MVC architecture
  • Data Transfer Objects for secure data transmission
  • Repository pattern for database operations
  • Service layer for business logic implementation
  • Minimal resource consumption, easy to host & scale
